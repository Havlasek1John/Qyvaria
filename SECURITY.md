# Security Policy

## Supported public repository

This public repository is for Qyvaria OS, Qyvaria AI Software, and the `qyvaria.py` source bundle.

## Reporting security issues

Please report security-sensitive issues privately to the maintainer.

Do **not** post the following in public GitHub issues:

- API keys or tokens
- private keys
- passwords
- personal data
- exploit details
- proprietary datasets
- non-public model credentials

## Public release safety checklist

Before every public release:

- Check that no private keys, API tokens, passwords, or secrets are present.
- Check that no personal data or private user data is included.
- Check that third-party license notices are preserved.
- Check that release ZIP files are intended for public distribution.
- Post checksums for release assets when possible.

## Experimental software notice

Qyvaria is experimental AI software. Users should review code and package contents before running it, especially when adding local credentials, network access, private files, or third-party extensions.
