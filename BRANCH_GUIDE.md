# GitHub Branch Guide for Qyvaria

Your public repository can use one main default branch for the public project.

Recommended:

```text
Qyvaria
```

Use that branch for:

- README.md
- LICENSE
- qyvaria.py
- SECURITY.md
- CONTRIBUTING.md
- docs
- examples
- scripts

Optional branches can be used later:

```text
Qyvaria-OS       desktop package work
Qyvaria-Kernel   kernel-only development
Qyvaria-Docs     documentation-only development
```

Note: The current branch name shown in GitHub may be `Qyvaria`. Upload the public repository files to the default branch first so visitors immediately see the README.
