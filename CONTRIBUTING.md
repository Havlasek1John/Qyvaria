# Contributing to Qyvaria

Contributions are welcome when they preserve the Qyvaria project identity, Apache-2.0 license, attribution, safety boundaries, and public-source structure.

## Before contributing

Please read:

- `README.md`
- `SECURITY.md`
- `CODE_OF_CONDUCT.md`
- `CREDITS.md`
- `THIRD_PARTY_NOTICES.md`

## Contribution checklist

- Keep attribution to **Jan Havlasek** for Qyvaria OS, Qyvaria AI Software, and `qyvaria.py`.
- Preserve the Apache-2.0 license notice.
- Preserve acknowledgement of **Jiri Burda** in project credits.
- Do not add secrets, API tokens, private keys, passwords, personal data, proprietary datasets, or private logs.
- Document new dependencies.
- Add or update third-party notices when needed.
- Prefer small, reviewable pull requests.
- Test startup on the platform affected by the change.
- Clearly label experimental features.

## Feedback

General feedback should go to Qyvaria Forums:

https://qyvaria.boards.net/

Repository-specific code or documentation issues can be opened on GitHub Issues.
