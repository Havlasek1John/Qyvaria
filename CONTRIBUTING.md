# Contributing to Qyvaria

Contributions are welcome when they preserve the Qyvaria project identity, license, attribution, and public safety boundaries.

## Contribution checklist

- Keep attribution to **Jan Havlasek** for Qyvaria OS, Qyvaria AI Software, and `qyvaria.py`.
- Preserve the Apache-2.0 license notice.
- Do not add secrets, API tokens, private keys, passwords, or private user data.
- Document new dependencies.
- Prefer small, reviewable pull requests.
- Test startup on the platform affected by the change.
- Keep large release binaries in GitHub Releases instead of the repository root.

## Development note

The main repository should stay clean and public-source friendly. Large Windows/Linux desktop package archives and large PDF libraries should usually be published through GitHub Releases.
